# 标记和解剖结构

安静、深思熟虑的外观是几个固定规范加上两个负空间。数据是唯一允许响亮的东西。

## 标记规范（每个图表固定）

| 标记 | 规范 |
|---|---|
| 条形/柱 | **≤ 24px 厚**（封顶它——永远不要填充槽位；让带的剩余部分成为空气）；**4px 圆角数据端，基线处为方形**；从单个基线生长 |
| 线 | **2px**，圆角连接/端点 |
| 标记/端点 | **≥ 8px**（r ≥ 4），用系列颜色填充 |
| 面积填充 | 系列色相在 **~10% 不透明度**（冲洗，绝不是饱和块） |
| 网格线/轴 | 比表面深一步的灰色，**极细线（1px），实线**（绝不虚线），后退 |

## 两个间隔符（白色做分隔）

- **表面间隙。** 表面颜色中的 **2px 间隙**分隔接触的标记——堆叠条形的每一段，以及每个相邻（接触）的条形，宽度相同。在堆叠中保持一个一致的宽度；相隔一步的邻居因为间隙而读作不同，而不是在它们周围绘制的描边。
- **表面环。** 点和端标记携带表面颜色中的 **2px 环**，以便它们在穿过线或相互重叠的地方保持清晰。环是标记的悬停/命中目标的一部分，不仅仅是间距——见 `interaction.md`（小点很容易为悬停而尺寸不足）。

永远不要在标记周围绘制边框来分隔它。间隙和环是机制；描边添加不是数据的数据权重墨水。

## 标签和图例

**两个或更多系列的图例始终存在**——可靠的身份通道；永远不要让读者仅依靠颜色匹配。直接标签然后骑在标记上以*补充*它。**单个系列不需要图例框**：只有一种颜色，所以图表的标题或副标题已经说明绘制的内容。带有一个样本的框重述标题并花费空间。

- **选择性标记——从不在每个点上都有数字。** 每个点或段旁边的值是混乱且不可读的。标记端点、极值或故事关于的系列；让轴、图例和工具提示/表格承载其余部分。直接标签工作*因为*它们是稀疏的——淹没图表，它们停止工作。
- **直接标签在网格线之前；网格线在第二个轴之前。**
- **不适合的标签不会被剪裁——先测量。** 仅当渲染的文本在两侧都有舒适的填充时，才将标签*放置*在条形或堆叠段*内部*。如果它不适合：对于整个条形/柱，将标签移到条形端之外（如果外面也没有空间，则移到工具提示）；对于*内部*堆叠段（没有自由端），跳过内联标签并让图例 + 工具提示承载它。无论哪种方式，值都保留在表格视图中，所以没有任何东西被阻止。永远不要在段上使用 `overflow: hidden` 来"解决"它——这会裁剪首/尾字符，并且比没有标签更糟糕。文本永远不会溢出或被其自己的标记剪裁。
- 条形 → 尖端的值。柱 → 帽上的值。线 → 端点的值。
- Y 轴刻度：四舍五入到干净的数字（0 / 1,000 / 2,000），千位逗号；它们承载你没有直接标记的值，所以保留它们，除非每个值都被标记。
- **文本从不佩戴数据颜色。** 标记——条形、线、点、面积填充——携带系列颜色；标签、值、图例和轴文本使用**文本标记**（主/次/静默）。浅分类色相（黄色、浅绿色）作为表面上的文本是不可读的。身份来自文本*旁边*的彩色标记——点、短线键、样本——绝不是来自着色文本本身。设置在彩色填充*内部*（堆叠段、地图瓦片）的标签是一个例外：根据填充的亮度选择白色或墨水，以便它始终清除对比度。
- **当端标签碰撞时，不要堆叠它们。** 直接端标签在系列在右边缘分离时工作。当线收敛时，垂直推开标签会将它们与线分离并读作噪音——而是使用**引导线**（从标签到线端的细连接器）、分面到**小倍数**，或回退到图例 + 工具提示。超过 ~4 个收敛系列，小倍数通常是正确的。

## 图形——当形式是数字时

- **统计磁贴**契约：`label`（句子大小写，没有尾随冒号）· `value`（无衬线半粗体，自动紧凑：1,284 / 12.9K / $4.2M）· `delta`（可选；有符号，与命名期间相比；颜色 = 方向 × 上升是否良好）· `trend`（可选；去强调色相中的 12 点迷你图，当前期间在强调色中）。
- **仪表：** 填充携带严重性（强调 → 警告 → 危险）；未填充轨道是**相同渐变的较浅步骤**（蓝色上的蓝色等），以便状态在整个条形上读取。
- **英雄图形。** 仪表板引导的单个数字，≥48px，使用与其他所有内容相同的无衬线字体（绝不是显示或衬线字体——它读作偏离品牌的装饰）。每个视图正好一个。
- **大数字的比例数字；仅在列中使用表格。** 大型独立值（英雄图形、统计磁贴值）使用字体的默认比例数字——`tabular-nums` 给每个数字 `0` 的宽度，所以像 `121` 这样的数字在显示尺寸下看起来松散。将 `font-variant-numeric: tabular-nums` 保留给必须垂直对齐的数字列（表格行、轴刻度）。

## 纹理——备份通道（可选）

色相失败的地方——完全严重 CVD、灰度打印、`forced-colors`——纹理承载身份。一个方向性手绘填充，仅在 **45° 及其 135° 镜像使用**（绝不是水平/垂直——那些读作网格线/条形）。色调上墨水（来自填充自己渐变的一步），跨槽位响度相等。在值标度上，纹理是*有序的*（旋转步骤随量级；臂角度携带发散符号），以便它从不误陈述值。由可访问性设置、打印或 `forced-colors` 触发——默认情况下从不开启。（见 `palette.md`。）
