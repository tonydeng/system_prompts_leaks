# 组件——图表的组成部分

图表由这些部分构建，用纯 HTML/SVG 组装。第 0 层是所有内容挂载的基础；系统层是使方法可移植的内容（并且，本身就是此技能）。

## 第 0 层——基础
- **颜色角色**——分类（8 × 明/暗）、顺序渐变、发散对、状态（4）、去强调/"其他"、灰度图表家具（轴/网格/标签/表面）。定义为 HTML 顶部的 CSS 自定义属性——见 `palette.md`。
- **纹理填充**——方向性填充 + 45°/135° 旋转。
- **图表容器**——一个拥有响应式大小、标题/说明和**表格视图切换**（每个图表的可访问性双胞胎）的 `<figure>`（或卡片 `<div>`）。**任何固定高度都包括 x 轴带**（绘图高度 + 轴标签），以便卡片永远不会获得嵌套垂直滚动；更喜欢让容器随其内容增长。
- **图例**（切换到隔离、纹理感知样本）· **工具提示** · **轴** · **数据标签**。

## 第 1 层——人们要求的图表
- **条形图**——分组 + 堆叠、细条默认、水平 + 垂直。
- **折线图**——多系列、软填充面积变体、可访问性标记。
- **统计磁贴**——值 + 增量 + 可选迷你图（图形契约）。
- **仪表/进度轨道**——相同渐变轨道。

## 第 2 层——完善工具包
- **面积图**（堆叠、带边缘 = 线）· **迷你图** · **热图**
- **比例图例**（顺序 / 发散）· **图表过滤器 / 时间范围** · **空状态**

## 系统层——成为技能
- **六检查验证器**——`scripts/validate_palette.js`（调色板验证）。
- **主题引擎**——将客户的渐变捕捉到通过值（color-formula.md）。
- **图表类型启发式**——选择形式（choosing-a-form.md）。
- **表格视图生成器**——任何图表的 WCAG 干净等效项。

注意：局部到整体依赖于堆叠条形图；圆环保持低优先级。小倍数是这些之上的布局模式，而不是单独的部分。如果散点密集表面落地，散点加入第 2 层。
