# 参考调色板

这是数据可视化方法的**参考实例**：方法需要的每个参数，都用经过验证的默认调色板填充。技能的其余部分是系统无关的——**要针对你的品牌，替换此文件的值**并重新运行验证器。没有其他改变。

## 如何使用这些值

下面的所有内容都是纯十六进制。在 HTML 图表中，**在文件顶部的本地 `<style>` 块中将你使用的槽位定义为 CSS 自定义属性**，然后在整个过程中按角色引用它们——以便明/暗值在一个地方交换，并且图表主体是针对角色而不是原始十六进制编写的：

```css
.viz-root {
  --surface-1:      #fcfcfb;   /* 图表表面 */
  --text-primary:   #0b0b0b;
  --text-secondary: #52514e;
  --series-1:       #2a78d6;   /* 分类槽位 1 */
  /* …仅此图表使用的角色 */
}
@media (prefers-color-scheme: dark) {
  .viz-root {
    --surface-1:      #1a1a19;
    --text-primary:   #ffffff;
    --text-secondary: #c3c2b7;
    --series-1:       #3987e5;
  }
}
```

## 分类调色板

两种模式都已选择。暗列是为暗表面步进的相同八个色相，而不是单独的调色板：

| 槽位 | 色相 | 浅色 | 暗色 |
|------|-----|-------|------|
| 1 | 蓝色 | `#2a78d6` | `#3987e5` |
| 2 | 浅绿色 | `#1baf7a` | `#199e70` |
| 3 | 黄色 | `#eda100` | `#c98500` |
| 4 | 绿色 | `#008300` | `#008300` |
| 5 | 紫色 | `#4a3aa7` | `#9085e9` |
| 6 | 红色 | `#e34948` | `#e66767` |
| 7 | 洋红色 | `#e87ba4` | `#d55181` |
| 8 | 橙色 | `#eb6834` | `#d95926` |

浅色模式最差相邻 CVD ΔE 为 24.2——远高于 ≥12 目标。三个浅色模式槽位（浅绿色、黄色、洋红色）在浅色表面上低于 3:1 对比度：**缓解规则**适用（发布可见的直接标签或表格视图）。暗色步骤是为暗带选择的（OKLCH L ≈ 0.48–0.67，在暗表面上 ≥ 3:1）并作为一组验证——最差相邻 ΔE 10.3，下限带，因此四个以上系列在暗模式下也依赖直接标签或纹理。

槽位**排序**是 CVD 安全机制，而不是装饰性的——它是通过枚举排序并选择最大化最小相邻 ΔE 的那个推导出来的（见 `color-formula.md` § 主题）。当你交换品牌的色相时，做同样的事情：对候选排序运行验证器并保留最好的。

## 顺序色相

默认单色相：**蓝色**，浅→深。当两个顺序上下文同时出现时，第二个采用下一个分类槽位的色相（浅绿色），每个作为自己的单色相渐变。

| 步骤 | 十六进制 | 步骤 | 十六进制 | 步骤 | 十六进制 | 步骤 | 十六进制 |
|---|---|---|---|---|---|---|---|
| 100 | `#cde2fb` | 250 | `#86b6ef` | 400 | `#3987e5` | 550 | `#1c5cab` |
| 150 | `#b7d3f6` | 300 | `#6da7ec` | 450 | `#2a78d6` | 600 | `#184f95` |
| 200 | `#9ec5f4` | 350 | `#5598e7` | 500 | `#256abf` | 650 | `#104281` |
| | | | | | | 700 | `#0d366b` |

完整的 100→700 范围用于**顺序**编码（连续量级——热图、等值线），其中最浅的步骤意味着"接近零"并被允许退向表面。对于**有序**渐变（离散有序标记——漏斗阶段、层级——使用 `--ordinal` 验证），最接近表面的步骤必须仍然清除 2:1：在浅色上，开始不浅于**步骤 250**（`#86b6ef`，2.06:1）；在暗色上，不超过**步骤 600**（`#184f95`，2.15:1）。

## 发散对

**蓝色 ↔ 红色**——读作相反的暖/冷极点。中性中点是灰色（浅色 `#f0efec`，暗色 `#383835`）。每臂相等步数。（蓝色↔浅绿色被拒绝——两者都冷，中点不读作"无"。）

## 状态调色板（固定——从不主题化）

| 角色 | 十六进制 | 浅色表面对比度 | 暗色表面对比度 |
|---|---|---|---|
| 良好 | `#0ca30c` | 3.27 | 5.19 |
| 警告 | `#fab219` | 1.79 | 9.49 |
| 严重 | `#ec835a` | 2.57 | 6.60 |
| 关键 | `#d03b3b` | 4.68 | 3.62 |

暗色：相同的四个步骤——在暗表面（`#1a1a19`）上都清除 3:1，并且与暗分类槽位保持不同。在浅色表面上，警告和严重按设计低于 3:1；**图标 + 标签**配对是缓解措施，因此状态颜色从不单独携带含义。这些步骤故意与分类槽位不同，以便状态颜色从不模仿系列。

## 纹理填充（可访问性通道）

一个手绘**"线条"**填充，仅在 **45° 及其 135° 镜像使用**。色调上墨水（填充自己渐变的较暗步骤）。在值标度上，它是*有序的*（旋转步骤随量级；臂角度携带发散符号）。由可访问性设置、打印或 `forced-colors` 触发——从不装饰，默认情况下从不开启。

## 表面（用于验证器）

- 浅色图表表面：`#fcfcfb`
- 暗色图表表面：`#1a1a19`

这些是验证器的内置默认值。**当你交换自己的调色板时，针对自己的表面重新运行：**
`--surface <your-light> --mode light` 和 `--surface <your-dark> --mode dark`——
对比度和带结果仅对图表实际渲染的表面有意义。

## 过滤器控件

过滤器是标准 UI，而不是图表组件——图表层仅在 `interaction.md` 中添加组合规则。日期范围控件是预设行列表（今天、过去 7/30/90 天、本月至今），选择由 16px 粗体复选标记标记，悬停为幽灵冲洗，自定义范围隐藏在页脚中的发丝后面。维度过滤器是标准组合框。

## 字体和数字

所有内容——包括英雄图形——都保持在系统无衬线中：`system-ui, -apple-system, "Segoe UI", sans-serif`。任何地方都没有显示或衬线字体。大型独立数字（英雄图形、统计磁贴值）使用默认比例数字；将 `font-variant-numeric: tabular-nums` 保留给必须垂直对齐的列（表格行、轴刻度）。在此替换你的品牌 UI 无衬线。
