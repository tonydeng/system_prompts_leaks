# 反模式——什么会出错

根据此列表检查每个图表。如果你的输出匹配一个条目，它就是错的——在发布之前修复它。这些是真实的失败模式，每个都在发布的仪表板中被捕获。

## 颜色和编码

**❌ 双轴图表（一个图上有两个 y 刻度）。**
为什么误导：两个刻度的对齐是任意的，因此图表发明了数据中不存在的相关性。真实示例：一个"采用"图表绘制用户（0–30k）与会话（0–800k）——审查员将其标记为看起来"幻觉"。
✅ 改为：两个图表、小倍数，或将两个系列索引到**一个**轴上的公共基数（t0 时 =100）。

**❌ 过滤时重新着色。** 按当前排名分配颜色，因此过滤掉系列会重新绘制幸存者。
为什么：学习"Acme 是蓝色"的读者现在被误导了。
✅ 颜色跟随实体，而不是其行号。幸存者保持其色相。

**❌ 超过 8 个循环/生成色相。** 第 9 个分类颜色，生成或重用。
为什么：在 CVD 下与现有槽位无法区分；破坏顺序检查。
✅ 将尾部折叠到"其他"、分面到小倍数，或使用复合编码。

**❌ 凭眼力判断色盲安全。** "这些看起来足够不同。"
✅ 运行 `scripts/validate_palette.js`。相邻 ΔE ≥ 12，或 8–12 带有次要编码。

**❌ 名义类别上的值渐变。** 当类别没有自然顺序（产品、团队、端点）时，将每个条形着色为越大越暗。
为什么：它将条形长度双重编码为色相，在图表已经显示的信息上烧掉唯一的自由通道，并且通过设计失败分类检查（渐变跨越亮度带并降至色度下限以下）。
✅ 一个系列→每个条形的一种颜色（槽位 1）。有序类别（漏斗、层级、年龄带）→有序渐变，使用 `--ordinal` 验证。

**❌ 彩虹/非相邻顺序。** 用于量级的多色相渐变。
✅ 一个色相，浅→深。（类似邻居或语义热是唯一的多色相顺序例外，始终带有比例图例。）

**❌ 发散中点的色相，或两个冷色相作为两个极点。**
为什么：中点必须读作"无"；极点必须读作相反。blue↔aqua 失败此（两者都冷）；blue↔red 或 blue↔orange 成功（暖/冷）。
✅ 读作相反的两个色相 + 中性灰中点。

**❌ 状态颜色用于非状态系列**（或系列颜色用于状态）。
✅ 仅当颜色*意味着*良好/不良时使用状态标记；身份时使用分类。

## 形式

**❌ 当故事是一个数字时使用八个分类色相。** 图表错过其观点的最常见方式。
✅ 强调（高亮一个，将其余变灰），或统计磁贴/英雄数字。

**❌ 单条形条形图，或 2 片饼图。**
✅ 统计磁贴。数字就是图表。

**❌ 用于比较接近值的圆环/饼图。**
✅ 条形或数字。仅一目了然的局部到整体，≤ 6 段。

**❌ 超过 ~7 个携带意义的颜色类别。**
✅ 表格或表格 + 图表。超过 ~7 个分箱，相邻类别模糊。

## 标记和镀铬

**❌ 厚饱和块、重网格线、没有呼吸空间。** 在规模上读作响亮，甚至幼稚。
✅ 细标记、极细后退网格/轴、充裕填充。饱和填充用于小标记和强调，从不是大块。

**❌ 虚线网格线或轴规则。** 虚线增加视觉噪音，当它只是网格时读作"投影"或"阈值"。
✅ 网格线和轴是实线极细线，比表面深一个色调。

**❌ 每个数据点上的数字。** 每个点或段旁边的值是混乱且不可读的。
✅ ≥ 2 个系列的图例始终存在；*选择性*直接标记（端点、极值、重要的一个系列），并让轴 + 工具提示承载其余部分。

**❌ 在标记周围绘制边框以分隔它们。**
✅ 填充之间的 2px 表面间隙（堆叠段和相邻条形一样）和 2px 表面环（在重叠标记上）。

**❌ 标签被过小的条形或堆叠段剪裁或溢出**——包括 `overflow: hidden` 裁剪段内标签的首/尾字符。
✅ 仅当标签带有填充适合时才在标记内渲染它；否则将其移到条形端之外，或将其丢弃到工具提示/图例（值保留在表格视图中）。

**❌ 图表容器的固定高度排除了 x 轴带**——绘图适合，轴标签不适合，因此卡片获得微小的嵌套垂直滚动。
✅ 调整容器大小以包括轴标签（绘图高度 + x 轴带），或让容器随其内容增长而不是固定高度。

**❌ 英雄图形上的显示或衬线字体。** 它读作偏离品牌的装饰。
✅ 英雄图形使用与其他所有内容相同的无衬线字体。

**❌ 大型独立数字上的 `tabular-nums`。** 等宽数字使 `121` 在显示尺寸下看起来松散。
✅ 英雄和统计磁贴值上的比例数字；`tabular-nums` 仅在数字垂直对齐的地方（表格行、轴刻度）。

**❌ 默认开启纹理，或作为装饰。** 密集的角度字段是前庭风险，并在值标度上读作噪音。
✅ 纹理是可选的（a11y 设置、打印、强制颜色），仅 45°/135°，在值标度上有序。

## 交互和可访问性

**❌ 工具提示作为读取值的唯一方式。**
✅ 工具提示增强，从不阻止——每个值也可通过直接标签或表格视图访问；键盘焦点显示与悬停相同的内容。

**❌ 精确悬停目标——你必须死中心落地的 8px 散点。**
✅ 命中区域包括 2px 间隙并满足 ~24px 最小值；密集散点使用最近点 / Voronoi 层。

**❌ 每图表过滤器，或图表卡内的过滤器。**
✅ 一个过滤器行位于其范围的所有内容上方；所有图表针对同一切片重新渲染。

**❌ 重新获取时的骨架闪烁。**
✅ 以降低的不透明度保持先前的渲染——没有布局跳跃。

**❌ 连续标度上没有表格视图/仅颜色编码。**
✅ 每个图表都有表格视图双胞胎（WCAG 干净的等效项）。
