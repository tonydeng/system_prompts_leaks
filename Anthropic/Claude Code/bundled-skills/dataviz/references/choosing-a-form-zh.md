# 选择形式

在颜色**之前**决定这一点。数据的工作选择形式——有时正确的形式不是图表。

## 它甚至是图表吗？

| 数据是… | 使用 | 不使用 |
|---|---|---|
| 单个当前值（+ 可能的趋势） | **统计磁贴**（值 + 增量 + 迷你图） | 单条形条形图 |
| 少量标题数字 | 统计磁贴的 **KPI 行** | 分组条形图 |
| 仪表板引导的单个数字 | **英雄图形**（≥48px，无衬线） | — |
| 针对限制的单个比率 | **仪表**（相同渐变轨道） | 2 片饼图 |
| 超过 ~7 个都携带意义的类别 | **表格**（或表格 + 图表） | 更多颜色 |

如果图表*是*正确的，按工作选择类型：

## 工作 → 类型

| 工作（读者必须做什么） | 默认形式 | 颜色工作 |
|---|---|---|
| 比较量级，低→高 | 条形/柱状；网格的**热图** | 顺序（一个色相） |
| 随时间趋势 | 线；单个系列的面积 | 顺序或 1 个分类 |
| 区分不同系列 | 分组/堆叠条形、多线 | **分类** |
| 一个系列是重点，其余是上下文 | **强调**（高亮一个，将其余变灰） | 1 个色相 + 灰色 |
| 基线之上/之下；到目标的 Δ | 发散条形，或线与基线 | 发散 |
| 局部到整体 | **堆叠条形**（对于许多/长名称类别，采用水平） | 分类 |
| 有序标度份额（Likert、情感、同意↔不同意） | **发散堆叠条形**，以中性为中心 | 发散 |
| 每项之前→之后 | 哑铃 | 1 个色相，2 个色调 |

## 表格背后的规则

- **顺序是安全的默认值。** 一个色相，越多越暗。它保持清晰和一致，并且很难误读。除非数据的工作专门是*身份*或*极性*，否则使用它。
- **分类用于当系列是主体时**——并且它有真正的成本：它可能埋没实际上重要的一个数据点。如果故事是"这一个上升了"，那是**强调**，而不是分类。
- **强调** = 最少使用的形式。一个系列在强调色相中，其余在去强调灰色中。通常是"让这个图表更清晰"的诚实答案。
- **纹理是可选表达，不是默认形式。** 它仅为了可访问性（完全 CVD）、打印/导出和 `forced-colors` 赢得它的位置。从不装饰。→ 见 `marks-and-anatomy.md`。

## 系列计数阶梯（分类）

| 系列 | 处理 |
|---|---|
| 1–3 | 仅颜色对每个人都舒适；直接标记 |
| 4 | CVD 下限进入——直接标签成为强制性的，而不是礼貌 |
| 5–6 | 软上限；图例或小倍数 |
| 7–8 | 标记上限；超过它，将尾部折叠到"其他"、分面到小倍数，或使用复合编码（色相 × 形状） |

永远不要通过生成更多色相来解决"太多系列"。生成的第 9 个色相在 CVD 下与现有色相无法区分，并且破坏每个检查。
