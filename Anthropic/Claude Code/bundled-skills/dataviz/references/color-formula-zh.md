# 颜色公式

颜色**不是手工挑选的**。每个图表颜色恰好做四个工作之一，并且调色板只有在通过六个检查时才合法。检查是产品——它们使调色板可以安全更改，并让相同的方法在任何设计系统的渐变上运行。

## 四个工作

| 工作 | 它编码的内容 | 结构 |
|---|---|---|
| **分类** | 身份（哪个系列） | 8 个色相，固定顺序，按顺序分配，从不循环 |
| **有序** | 序列中的位置（漏斗阶段、层级、分箱） | 一个色相，单调亮度步骤；浅端仍然在表面上 ≥ 2:1 |
| **顺序** | 量级（多少） | 一个色相，步骤 100→700，浅→深；在暗模式下翻转锚点 |
| **发散** | 极性（基线的哪一侧） | 两个色相 + 中性灰中点；每臂相等步骤 |
| **状态** | 状态（良好→关键） | 小固定标度，保留含义，始终图标+标签 |

**分类还是有序？** 如果交换类别顺序会改变含义——漏斗阶段、大小层级（S/M/L）、年龄带、队列分箱——它是**有序**并采用单色相渐变，以便读者在颜色中看到顺序。如果交换不会——产品名称、团队、地区、端点——它是名义分类，每个条形采用*相同*的槽位 1 色相（一个系列，所以没有图例框——标题命名它），或者当有 N 个独立系列时采用槽位 1..N。永远不要按值着色名义条形：这花费身份通道重新编码条形长度已经显示的内容。

## 六个检查

每个分类颜色——当前或提议的——必须通过所有六个。

1. **固定色相锚点。** 固定顺序的八个族。顺序是 CVD 安全机制；它从不改变。*（结构——强制执行，而不是测量）*
2. **每种模式的亮度带。** OKLCH L ≈ 0.43–0.77 浅色；≈ 0.48–0.67 深色。*（验证器）*
3. **色度下限。** OKLCH C ≥ ~0.10——低于它，色相读作灰色并停止做身份工作。*（验证器）*
4. **CVD 分离。** Machado-2009 ΔE ≥ 12 目标 / ≥ 8 下限（下限仅在次要编码时合法），在红色盲和绿色盲下。堆叠/条形/线的*相邻*对（只有邻居接触——分配从不跳过）；**散点、气泡、等值线和小倍数的所有对**，其中任何两个标记可以并排坐——在那里传递 `--pairs all`，否则真正的折叠保持隐藏。*（验证器）*
5. **与表面的对比度。** 标记 ≥ 3:1；在值可以通过另一种方式读取的地方有条件地放宽（可见标签或表格视图）。*（验证器）*
6. **仅记录的调色板。** 每个槽位都是实例文件（`palette.md` 或其等效项）中的十六进制——没有凭眼力的值。*（结构；对于客户的渐变，捕捉到最近的——见下文）*

## 运行检查——永远不要凭眼力判断它们

```
node scripts/validate_palette.js \
  "#2a78d6,#1baf7a,#eda100,#008300,#4a3aa7,#e34948,#e87ba4,#eb6834" --mode light
```

（`scripts/` 相对于此技能的基础目录，显示在提示的顶部。）

（或将其作为 `<script type="module">` 加载到图表自己的页面中——它从 `<body>` 读取 `data-palette` 并记录 `console.table` 报告）

报告每个可计算的检查（2–5）带有 PASS / WARN / FAIL 以及最差的 CVD 对。退出 0 = 没有硬 FAIL（WARN 带——下限带 CVD 8–12 和次 3:1 对比度缓解——仍然退出 0 并需要次要编码）；任何 FAIL 退出 1。每种模式运行一次（`--mode dark --surface "#1a1a19"`），并为散点/气泡/地图/小倍数图表添加 `--pairs all`（其中任何两个标记都可以是邻居——默认相邻检查会隐藏折叠）。对于**有序**渐变，传递 `--ordinal`——它切换到渐变检查（单调 L、相邻 ΔL ≥ 0.06、浅端对比度 ≥ 2.0:1、单色相）而不是分类六个。CVD 上的 WARN（8–12 下限）**仅**在你也发布次要编码（直接标签、间隙或纹理）时才合法。对比度上的 WARN **不可忽略**——它强制缓解通道（可见直接标签或表格视图）；发布次 3:1 填充而没有其中任何一个是失败。

**范围——验证器做什么和不涵盖什么。** 这六个检查验证*分类*调色板（系列身份）。它们**不**判断单独的状态/文本颜色或顺序渐变。对于单个状态或文本颜色，运行 WCAG *文本*对比度检查（4.5:1 正常，3:1 大）——`validate_palette.js` 正是为了此导出 `contrast(a, b)`。对于顺序/发散，检查是渐变上的亮度单调性，而不是相邻 CVD——在顺序渐变上运行分类验证器**将按设计失败**（它跨越带；步骤坐得很近），这是预期的，不是真正的失败；不要"修复"一个好的渐变来满足它。

## 捕捉到通过（任何设计系统）

给定客户的渐变和所需顺序：
1. 对于每个槽位，选择其 OKLCH L 位于模式带中且 C ≥ 下限的步骤。
2. 运行验证器。对于任何低于 ΔE 12 的相邻对，将一个槽位微调 ± 一个步骤（保持其色相，移动其亮度）并重新运行。
3. 重复直到最差的相邻对清除下限。功能保留，客户的色相保持。

## 主题

槽位**顺序**是一个可分离的、命名的选择——一个*主题*——在相同的色相和相同的六个检查上。每个设计系统命名一个默认顺序和任何替代；交换主题调整情绪而不触及方法。表面采用一个主题并冻结它；永远不要在仪表板内混合主题。（见 `palette.md`。）

**当系统还没有主题时推导顺序：**不要猜测。枚举系统色相的候选顺序，对每个运行验证器，并选择最大化*最小相邻* CVD ΔE 的那个。（通过色相族类比从已知良好的顺序播种，然后优化，是可以的——这正是 `palette.md` 中默认值的推导方式。）

## 状态是固定的

状态从不跟随主题——它是一个小固定标度（良好 → 警告 → 严重 → 关键），具有保留含义，在故意与分类槽位不同的步骤上，以便状态颜色从不模仿系列，并且始终与图标+标签配对（在浅表面上，警告和严重按设计低于 3:1——配对是缓解）。（`palette.md` 中的确切步骤。）冲突规则：当系列*意味着*良好/不良（错误率、通过/失败）时，它佩戴状态标记；当它只是"系列 4"时，它佩戴分类——永远不要在一个图表中两者都有。
