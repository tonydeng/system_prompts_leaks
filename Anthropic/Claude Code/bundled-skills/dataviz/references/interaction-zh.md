# 交互——工具提示和过滤器

HTML 图表默认是交互式的——悬停层是交付物的一部分，而不是升级。省略它是例外（裸统计磁贴），绝不是默认。用与静态渲染相同的关心来设计它。

## 工具提示和悬停

工具提示**增强，从不阻止**：工具提示显示的每个值也可以在没有它的情况下通过直接标签或表格视图访问。键盘焦点上的详细信息与悬停相同。

- **十字准线找到 X。** 垂直发丝跟踪指针并捕捉到最近的数据位置。读者瞄准日期，从不瞄准 2px 线。
- **在条形和单元格上，标记是命中目标。** 没有十字准线——每个条形、段、点或热单元格携带自己的 `pointermove`/`focus` 工具提示，显示类别和值，并且悬停的标记抬起（轻微变亮或轮廓），以便读者看到它响应。
- **一个工具提示，每个系列。** 读出列出该 X 处的每个系列——指针永远不必落在线或填充上以获取值。
- **标签是不可信的数据——使用 `textContent`。** 系列和类别名称通常来自 CSV 标题、工具输出或 API 响应。使用 `textContent` 或 `createTextNode` 将它们插入到工具提示/图例/表格 DOM 中，永远不要通过 `innerHTML` 字符串连接。
- **值领先，标签跟随。** 在工具提示中，值是强元素、高对比度元素，系列名称是次要的——图例的层次结构反转，因为在这里读者拥有系列并想要数字。
- **线条键，而不是框。** 工具提示行用系列颜色的短笔画键入其系列；在工具提示密度下，填充框是做标签工作的数据权重墨水。（图例仍然镜像标记：条形/区域的矩形，线的线条。）
- **命中目标大于标记。** 标记的悬停/焦点区域包括其 2px 表面间隙以及一些——永远不仅仅是绘制的像素。8px 散点是没有人可靠命中的精确点；给每个点至少 **24px** 的透明命中区域，或者——对于密集散点——最近点 / Voronoi 层，以便指针只需要*最近*，而不是死中心。（十字准线已经为线和条形图上的 X 执行此操作；散点和气泡需要每点版本。）
- **推离其标记的值生活在工具提示中。** 当标签不适合小条形内（见 `marks-and-anatomy.md`）时，该条形的命中区域在悬停和焦点上携带值——工具提示是其溢出之家，表格视图使其完全无需悬停即可访问。

## 过滤器和时间范围

每个监控仪表板都需要相同的控件。这些是**标准 UI，而不是图表标记**——用样式匹配图表镀铬的普通 HTML 表单控件构建它们。Dataviz 只添加组合规则：

- **一行，在图表上方。** 过滤器位于它们范围的内容上方的单个左对齐行中——永远不在图表卡内，永远不是每图表。如果一个图表需要自己的范围，它是一个不同的仪表板。
- **日期范围优先。** 它是每个读者都伸手可及的过滤器；预设（今天、过去 7 / 30 / 90 天）在自定义范围之前。
- **过滤器范围它们下面的所有内容。** 每个图表、统计和表格针对同一切片重新渲染，因此数字始终一致。
- **重新获取保持帧。** 当数据重新加载时，图表以降低的不透明度保持其先前的渲染——没有骨架，没有布局跳跃，没有闪烁。

一个好的日期选择器将预设列为行（没有人为了"过去 30 天"而与日历网格搏斗），用 16px 粗体复选标记标记选择，保持悬停为幽灵冲洗，以便它从不与选择竞争，并将自定义范围隐藏在页脚中的发丝后面。（见 `palette.md` 的参考规范。）
