# 数据可视化

图表由**人阅读，由你执行**。此技能将"让它看起来好看"转化为带有检查的过程，因此结果通过构造而非品味来保证正确。

**这里的方法是设计系统无关的。** 过程、形式启发式、六个检查或标记规范中没有任何内容特定于某个产品。设计系统提供一小组*参数*（其渐变、分类顺序、发散对、状态调色板、纹理、其表面、其过滤组件）；方法按原样使用它们。**经过验证的默认调色板**是参考实例，在 `references/palette.md` 中完全指定。要针对你的品牌，读取该文件的结构并替换其值——不要触及任何其他内容。

> 最重要的单一习惯：**颜色部分是可计算的，所以要计算它。**
> 永远不要凭眼力判断调色板是否对色盲安全——运行 `scripts/validate_palette.js`。

## 过程——按顺序执行这些

颜色放在最后。大多数糟糕的图表首先选择颜色。

1. **选择形式。** 数据的工作是什么——量级、身份、极性、单个标题、随时间变化？工作选择图表类型，有时答案*不是图表*（统计磁贴或英雄数字）。→ `references/choosing-a-form.md`
2. **按其工作分配颜色。** 分类（身份）、顺序（量级）、发散（极性）或状态（状态）——每个都有一个规则。按固定顺序分配分类色相，从不循环。→ `references/color-formula.md`
3. **验证调色板——运行脚本，不要推理 ΔE。**
   `node scripts/validate_palette.js "<hex,hex,…>" --mode light`（相对于此技能的基础目录——或将其作为 `<script type="module">` 加载到图表自己的页面中，在那里它从 `<body>` 读取 `data-palette` 并记录 `console.table` 报告）。它返回亮度带、色度下限、相邻对 CVD 分离和对比度的通过/失败。修复任何失败的内容后再继续。使用该模式的表面为 `--mode dark` 重新运行。
4. **应用标记规范和间隔符。** 细标记、锚定到基线的 4px 圆角数据端、2px 线条、≥8px 标记、填充之间的 2px 表面间隙（堆叠段和相邻条形一样）和重叠标记上的 2px 表面环、选择性直接标签。→ `references/marks-and-anatomy.md`
5. **添加悬停层——默认情况下。** HTML/SVG 图表*是*交互式的；在线/区域上发送十字准线+工具提示，在条/点/单元格上发送每个标记的悬停工具提示。唯一跳过它的形式是没有绘图的裸统计磁贴。命中目标大于标记；过滤器在图表上方的一行中。→ `references/interaction.md`
6. **最终可访问性传递。** 对于 ≥ 2 个系列，图例始终存在且 ≤ 4 个也直接标记（单个系列不需要图例框——标题命名它），因此身份从不仅是颜色；存在表格视图；暗模式是**选择的**——来自相同渐变的自己的步骤，针对暗表面验证，而不是自动翻转；纹理可用于 CVD/打印/强制颜色情况。
7. **渲染它并查看它。** 验证器检查颜色，而不是布局——打开或截图输出，并在称其完成之前凭眼力检查标签冲突、几何形状和溢出。

然后根据 **`references/anti-patterns.md`** 检查结果——它是错误目录。如果你的图表匹配一个条目，它就是错的。

## 不可协商（在每个设计系统中都为真）

- **按固定顺序分配分类色相，从不循环。** 第 9 个系列绝不是生成的色相——它折叠到"其他"、小倍数或复合编码中。
- **一个轴。** 从不是双轴图表（两个 y 刻度）。不同刻度的两个度量→两个图表、小倍数或索引到公共基数。*（这是 #1 图表错误——参见反模式。）*
- **颜色跟随实体，从不跟随其排名。** 更改系列计数的过滤器不得重新绘制幸存者。
- **顺序 = 一个色相，浅→深。发散 = 两个色相 + 中性灰中点。** 从不是彩虹；从不是发散中点的色相。
- **在发布任何分类调色板之前运行验证器。** CVD ≥ 12 是目标；8–12 是仅在次要编码时合法的下限。对比度 WARN 强制可见标签或表格视图——它不可忽略。
- **细标记；≥ 2 个系列的图例始终存在（一个系列没有），带有选择性直接标签（从不在每个点上都有数字）；后退网格/轴。**
- **文本使用文本标记，从不使用系列颜色**——值、标签和图例保持在主/次/静默墨水中；它们旁边的彩色标记承载身份。
- **状态颜色是保留的**（良好/警告/严重/关键），从不重用于"系列 4"；它们随图标+标签一起发布，从不仅颜色。

## 插入设计系统

方法是不变的；只有这些参数每个系统都会改变。参考实例——每个填充的值——是 `references/palette.md`。

| 参数 | 系统提供的内容 |
|---|---|
| **渐变** | 调色板从中提取的色相标度（命名步骤） |
| **分类主题** | 固定色相顺序（命名主题）；默认 + 替代 |
| **顺序色相** | 量级的默认单色相 |
| **发散对** | 两个暖/冷极 + 中性中点 |
| **状态调色板** | 良好 / 警告 / 严重 / 关键——与分类不同的步骤 |
| **纹理填充** | 一个方向性手绘填充，在 45° / 135° 使用 |
| **表面** | 明暗图表表面颜色（验证器需要这些） |
| **过滤控件** | 日期范围和维度控件（`interaction.md` 中的行为规范） |

要加入新系统：填充这些行，将其渐变提供给验证器，并让它将每个槽位捕捉到最近的通过步骤。结构和规则保持原样。
