# `Glob` 工具

一个快速的**文件名 / 路径匹配**工具。它回答这个问题：*"哪些文件的路径匹配此模式？"*

Glob 搜索文件**名称和路径** — 它**不**查看文件内部。（它的兄弟工具 `Grep` 搜索文件**内容**。见 `grep-tool.md`。）

---

## 何时使用它

- 按名称或扩展名定位文件（例如所有 `*.test.ts` 文件）。
- 在深入之前发现代码库的结构/布局。
- 查找匹配模式的最近修改的文件。
- 任何时候你会在终端中到达 `find . -name ...` 或 shell globbing。

> **重要：**对于文件发现，优先使用此工具而不是通过 Bash 工具运行 `find` 或 `ls`。它更快，适用于任何代码库大小，并返回有用的排序结果。

---

## 参数

| 参数 | 类型 | 必需 | 描述 |
|---------|------|--------|----------------------------|
| `pattern` | string | **是** | 用于匹配文件的 glob 模式（例如 `**/*.js`、`src/**/*.{ts,tsx}`）。 |
| `path` | string | 否 | 要在其中搜索的目录。默认为当前工作目录。**不要**传递 `undefined` 或 `null` 来表示"当前目录" — 只需省略参数。 |

---

## Glob 模式语法

Glob 模式比正则表达式更简单。最有用的标记：

| 标记 | 含义 |
|-----------|-----------------------------------------------------------------------|
| `*` | 匹配除路径分隔符（`/`）之外的任意字符运行。 |
| `**` | 递归匹配任意数量的目录（包括零个）。 |
| `?` | 匹配恰好一个字符。 |
| `{a,b,c}` | 匹配逗号分隔的替代项中的任何一个（大括号扩展）。 |
| `[abc]` | 匹配集合中的一个字符。 |

### 常见模式

- `*.md` — 搜索目录中的每个 Markdown 文件（仅顶层）。
- `**/*.md` — **任何**深度的每个 Markdown 文件。
- `src/**/*.js` — `src/` 下任何地方的所有 `.js` 文件。
- `**/*.{ts,tsx}` — 所有 TypeScript 和 TSX 文件，递归。
- `**/test_*.py` — 任何深度以 `test_` 开头的所有 Python 文件。
- `?[0-9]*.log` — 第二个字符是数字的日志文件。

---

## 输出

- 返回匹配文件路径的**列表**。
- 结果**按修改时间排序**，最近修改的在前。这很方便：你刚刚触摸的文件（或与最近工作最相关的文件）倾向于出现在顶部。
- 如果没有匹配，返回空结果（不是错误）。

---

## 示例

**项目中任何地方的所有 TypeScript 文件：**
```json
{ "pattern": "**/*.ts" }
```

**特定目录下的测试文件：**
```json
{
  "pattern": "**/*.test.{ts,tsx}",
  "path": "packages/web/src"
}
```

**仓库根目录的配置文件：**
```json
{ "pattern": "*.{json,yml,yaml,toml}" }
```

**每个 README，无论深度：**
```json
{ "pattern": "**/README.md" }
```

## Glob vs. Grep — 使用哪个？

| 你想要… | 使用 |
|------------------------------------------------- | -------- |
| 查找**命名**为某种方式的文件 | `Glob` |
| 查找**包含**某些文本的文件 | `Grep` |
| 列出所有 `*.py` 文件 | `Glob` |
| 查找哪些 `*.py` 文件调用 `requests.get` | `Grep`（带有 `glob: "*.py"`） |
| 理解目录布局 | `Glob` |
| 读取匹配的代码行 | `Grep`（`output_mode: "content"`） |

一个常见的工作流是**链接它们**：使用 `Glob` 缩小到一组候选文件，然后 `Grep` 搜索它们的内容。注意 `Grep` 也接受自己的 `glob` 参数，所以对于简单的"搜索这些文件类型"情况，你可能根本不需要单独的 `Glob` 调用。

---

## 提示和 gotchas

- **`*` 不跨越目录边界；`**` 跨越。** `src/*.js` 匹配 `src/` 中的文件，而 `src/**/*.js` 匹配 `src/` 和每个子目录中的 `.js` 文件。
- **省略整个项目的 `path`。**不要传递空字符串或占位符 — 只需省略它以从工作目录搜索。
- **按最近排序。**因为结果按修改时间排序，`Glob` 是回答"我最近更改了什么匹配 X 的东西？"的快速方法。
- **对于广泛的、开放式的搜索**跨越许多目录和命名约定，考虑委托给探索智能体而不是发出许多单独的 `Glob` 调用。
